
- **Goal**: Complete the implementation for the Citybus API route information in the bus alarm system.
- **Challenge**: Discovered that the Citybus API has a different structure and access method compared to the KMB API, leading to "Invalid/Missing parameter(s)" errors.
- **Solution**:
    - Updated `find_stops.js` and `index.js` to be operator-aware, supporting `kmb`, `ctb`, and `nwfb`.
    - Implemented operator-specific logic, using the known working method for KMB and adding robust error handling for others like Citybus.
    - The main server now accepts a `BUS_OPERATOR` environment variable to easily switch between bus companies.
    - The system now provides clear feedback to the user when an operator's API cannot be accessed as expected, preventing crashes and improving user experience.
- **Outcome**: The bus alarm system is now more flexible and resilient. While full Citybus integration is not complete due to the API differences, the groundwork has been laid to easily add support once the correct API access method is determined. The system can successfully monitor KMB routes and gracefully handles the current limitations with other operators.
