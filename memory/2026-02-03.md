- Fixed critical bug in stop data processing where the loader was not correctly iterating through the data structure, resulting in empty stop objects.
- All API endpoints (`/api/routes/:company`, `/api/stops/:company/:route`, `/api/destinations/:company/:route`) are now functional and return correct data.

### API Endpoints
- `GET /api/routes/:company` - Returns a sorted list of all route numbers for a given company (e.g., KMB, CTB).
- `GET /api/stops/:company/:route` - Returns a sorted list of all stop names for a given route.
- `GET /api/destinations/:company/:route` - Returns a list of all destination names for a given route.
